# Hodor Web React 迁移设计

## 目标

将 Hodor Web 从 Vue 3 渐进迁移到 React 18、TanStack Router 和 shadcn/ui。迁移期间保留现有 Vue 运行版，后端接口、Pancat 登录、Socket.IO 通道和生产数据合同保持兼容。首批交付 React 主壳、登录、项目入口和 3D 导演台，后续按业务页面逐步替换。

## 方案选择

采用同仓库双入口迁移。仓库新增 `src-react`、`index.react.html` 和独立 Vite 配置，提供 `dev:react`、`test:react`、`build:react` 命令。Vue 源码继续由现有命令维护，React 页面通过相同的 Hodor 后端 API 工作。完成关键页面迁移后，再把默认构建入口切换到 React，并归档 Vue 文件。

暂不采用微前端和 iframe。微前端会引入运行时编排、共享依赖和样式隔离问题；iframe 会增加登录态、主题和素材传递成本。3D 导演台本身是 React 工程，代码以保留来源声明的内部包形式并入仓库，由 React 路由直接渲染。

## 数据与登录

React 客户端沿用 `http://localhost:10588/api` 作为本地默认地址，并支持环境变量覆盖。登录请求继续调用 `POST /login/login`。成功后保留现有 `token`、`userId` 和 `pancatAccount` 三个浏览器存储键，确保迁移期间 Vue 与 React 可以共享会话。所有请求统一经过一个轻量 API 客户端；收到 401 时清理会话并跳转登录页。

浏览器存储只保存会话和界面偏好。项目、资产、分镜和 3D 工程快照由后端合同负责。3D 导演台首版允许本地草稿，但保存动作必须上传工程 JSON；截图先上传素材服务，取得 URL 或素材回执后再绑定分镜。

## 页面与视觉

视觉基调：深色制作台、冷灰层级、Pancat 蓝作为唯一主要强调色。首屏直接展示工作区，不增加营销区块。左侧导航负责项目、资产、分镜、生产和 3D 导演台；顶部只保留当前项目、运行状态和账号入口；中间区域交给业务页面。

登录页保持简洁，提供账号、密码、后端地址和明确错误提示。3D 导演台占满可用工作区，不套额外卡片。页面切换使用短距离淡入，任务状态变化使用克制的颜色与进度反馈。

## 迁移顺序

1. 建立 React 工具链、基础样式、路由和测试。
2. 移植 Pancat 登录、会话保护和 API 客户端。
3. 建立工作台主壳与项目入口。
4. 把 3D 导演台作为原生 React 路由接入，并连接项目与分镜合同。
5. 迁移资产、分镜、生产智能体和视频工作台。
6. React 达到功能对等后切换默认构建，归档 Vue 代码。

## 验收

- `yarn dev:react` 可以独立启动。
- 未登录访问受保护页面会进入登录页。
- Pancat 账号登录后可以进入项目和 3D 导演台。
- React 构建不依赖 Vue 类型检查，测试与构建命令可独立通过。
- 3D 截图能够上传并回写到指定项目或分镜。
- LICENSE、NOTICES 和 3D 导演台来源声明保留完整。
